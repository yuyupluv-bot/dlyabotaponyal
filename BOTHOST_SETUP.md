# BotHost: VK-бот и админ-панель из одного архива

Один и тот же архив загружается в **два отдельных приложения BotHost**. Оба приложения используют одну PostgreSQL-базу и один токен сообщества VK, но запускаются разными файлами.

## Приложение 1 — VK-бот

- Тип: постоянный Python-процесс / worker.
- Версия Python: 3.11.
- Установка: `pip install -r requirements.txt`
- Файл или команда запуска: `python main.py`
- Переменные: возьмите из `.env.bot.example`.
- HTTP-порт и домен этому процессу не нужны.

## Приложение 2 — админ-панель

- Тип: Web / HTTP Python-приложение. Обычный worker без публичного HTTP-домена не подойдёт.
- Версия Python: 3.11.
- Установка: `pip install -r requirements.txt`
- Файл или команда запуска: `python admin_main.py`
- Переменные: возьмите из `.env.admin.example`.
- `admin_main.py` сам запускает Gunicorn на порту из переменной `PORT`.
- Health check: `/health`.

Если в BotHost нет типа Web/HTTP, привязки домена или выдачи публичного URL, админ-панель там не откроется: тогда для панели нужен тариф/раздел BotHost с веб-приложениями либо другой HTTP-хостинг.

## Общие значения

В обоих приложениях должны **полностью совпадать**:

- `DATABASE_URL`
- `VK_TOKEN`
- `VK_GROUP_ID`

Не создавайте отдельную базу для админки: иначе изменения текстов, блокировки и заявки не будут видны боту.

## Первый запуск

1. Создайте PostgreSQL и получите внешний `DATABASE_URL`.
2. Создайте приложение бота, добавьте переменные из `.env.bot.example`, но пока не запускайте два экземпляра одновременно.
3. В консоли приложения один раз выполните:
   ```bash
   alembic upgrade head
   python -m scripts.seed
   ```
4. Запустите приложение бота командой `python main.py` и убедитесь, что Long Poll запущен без ошибок.
5. Создайте второе Web/HTTP-приложение, загрузите тот же архив, задайте переменные из `.env.admin.example` и запустите `python admin_main.py`.
6. Откройте выданный BotHost HTTPS-адрес. Проверка `https://ВАШ-ДОМЕН/health` должна вернуть `{"status":"ok"}`.
7. Войдите с `ADMIN_LOGIN` и `ADMIN_PASSWORD`.
8. Откройте «Тексты бота»: каталог синхронизируется с текущей версией.

## Важные ограничения

- Должен работать только **один экземпляр** `python main.py`, иначе два Long Poll процесса начнут обрабатывать одни события.
- Админ-приложение запускает только `admin_main.py`; не запускайте в нём `main.py`.
- Миграции выполняйте один раз перед первым общим запуском. При последующих обновлениях — один раз из любого одного приложения.
- Не кладите рабочий `.env` с паролями внутрь ZIP. Добавляйте секреты через настройки окружения BotHost.
- Для публичной панели оставьте `FLASK_COOKIE_SECURE=1` и используйте только HTTPS.

## Обновление

Остановите оба приложения, загрузите новый одинаковый архив в оба, один раз выполните `alembic upgrade head`, затем сначала запустите бот, потом админ-панель.
